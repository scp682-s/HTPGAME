## 🔒 安全防范措施

### 1. API密钥保护

**✅ 正确做法**：
```bash
# 在 Vercel 控制台配置环境变量
DEEPSEEK_API_KEY=sk-你的密钥
```

**❌ 错误做法**：
```javascript
// 不要在前端代码中硬编码API密钥
const API_KEY = "sk-9ab4e3c8af6241e5917f87d2c1ce651f"; // 危险！
```

### 2. .gitignore 配置

确保 `.gitignore` 包含：

```
# 环境变量
.env
.env.local
.env.*.local

# Python
__pycache__/
*.py[cod]
*$py.class
*.so

# Vercel
.vercel

# IDE
.vscode/
.idea/
*.swp
*.swo

# 操作系统
.DS_Store
Thumbs.db
```

### 3. 图片来源验证（已实现）

后端代码已经实现了严格的验证：

```python
# 白名单验证
ALLOWED_IMAGES = [
    'photo/1.png',
    'photo/2.png',
    'photo/3.jpg',
    'photo/4.jpg'
]

# 拒绝用户上传图片
if image_source.startswith("data:image") or image_source.startswith("blob:"):
    return jsonify({"error": "不支持分析用户上传的图片"}), 403
```

### 4. CORS 配置

当前配置允许所有来源，部署后应该限制：

```python
# 修改 backend/app.py
from flask_cors import CORS

# 限制只允许你的域名
CORS(app, origins=[
    "https://your-project.vercel.app",
    "http://localhost:*"  # 开发环境
])
```

### 5. 请求频率限制

添加速率限制防止滥用：

```python
# 安装 flask-limiter
pip install flask-limiter

# 在 app.py 中添加
from flask_limiter import Limiter
from flask_limiter.util import get_remote_address

limiter = Limiter(
    app=app,
    key_func=get_remote_address,
    default_limits=["100 per day", "10 per hour"]
)

@app.route('/api/generate-report', methods=['POST'])
@limiter.limit("5 per minute")  # 每分钟最多5次
def generate_report():
    # ...
```

### 7. 错误处理（已实现）

不要暴露敏感信息：

```python
except Exception as e:
    app.logger.error(f"生成报告时出错: {str(e)}")
    # 不要返回详细的错误堆栈给用户
    return jsonify({
        "error": "生成报告失败",
        "message": "服务器内部错误"  # 通用错误信息
    }), 500
```

## 🚨 如果 API 密钥泄露

1. **立即撤销旧密钥**
   - 登录 DeepSeek 控制台
   - 删除泄露的 API Key

2. **生成新密钥**
   - 创建新的 API Key
   - 更新 Vercel 环境变量

3. **检查使用记录**
   - 查看是否有异常调用
   - 确认没有产生额外费用

---

## 💡 最佳实践

1. **永远不要**在前端代码中存储 API 密钥
2. **使用环境变量**管理所有敏感信息
3. **定期轮换** API 密钥
4. **监控使用量**，设置告警
5. **限制请求频率**，防止滥用
6. **记录日志**，但不记录敏感信息
7. **使用 HTTPS**，保护数据传输
