# 前后端分离部署实施指南

## 架构说明

- **前端**：部署在 Vercel（全球 CDN 加速）
- **后端**：部署在腾讯云香港服务器（无需备案）
- **通信方式**：Vercel 反向代理到腾讯云后端

## 部署步骤

### 第一步：在腾讯云服务器上部署后端

#### 1.1 连接到腾讯云服务器

```bash
ssh root@你的服务器IP
```

#### 1.2 安装 Node.js（如果还没安装）

```bash
# 安装 Node.js 18.x
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证安装
node -v
npm -v
```

#### 1.3 上传项目代码

方式一：使用 Git（推荐）
```bash
# 在服务器上克隆项目
cd /home
git clone https://github.com/你的用户名/你的仓库名.git
cd 你的仓库名
```

方式二：使用 SCP 上传
```bash
# 在本地电脑运行
scp -r "c:\Users\redking\Desktop\大创项目\心宇宙重塑：房树人图像趣测" root@你的服务器IP:/home/
```

#### 1.4 安装依赖并启动服务

```bash
# 安装依赖
npm install

# 创建 .env 文件
nano .env
```

在 `.env` 文件中添加：
```
OPENAI_API_KEY=你的DeepSeek_API密钥
ADMIN_PASSWORD=你的管理员密码
```

保存并退出（Ctrl+X，然后 Y，然后 Enter）

#### 1.5 使用 PM2 保持服务运行

```bash
# 安装 PM2
npm install -g pm2

# 启动服务
pm2 start server.js --name "htp-backend"

# 设置开机自启
pm2 startup
pm2 save

# 查看服务状态
pm2 status
pm2 logs htp-backend
```

#### 1.6 配置防火墙

```bash
# 开放 3001 端口
sudo ufw allow 3001
sudo ufw status
```

#### 1.7 测试后端是否正常运行

```bash
curl http://localhost:3001/api/puzzle/games
```

如果返回错误信息（而不是连接失败），说明后端正常运行。

### 第二步：更新 Vercel 配置

#### 2.1 修改 vercel.json

**重要：将 `你的腾讯云服务器IP` 替换为实际的公网 IP**

当前配置：
```json
{
  "version": 2,
  "rewrites": [
    {
      "source": "/api/:path*",
      "destination": "http://你的腾讯云服务器IP:3001/api/:path*"
    }
  ],
  "env": {
    "ADMIN_PASSWORD": "123456"
  }
}
```

例如，如果你的服务器 IP 是 `43.154.123.45`，则改为：
```json
{
  "version": 2,
  "rewrites": [
    {
      "source": "/api/:path*",
      "destination": "http://43.154.123.45:3001/api/:path*"
    }
  ],
  "env": {
    "ADMIN_PASSWORD": "123456"
  }
}
```

#### 2.2 提交并推送到 Git

```bash
git add vercel.json
git commit -m "配置 Vercel 反向代理到腾讯云后端"
git push origin main
```

#### 2.3 Vercel 会自动重新部署

访问 https://vercel.com/你的项目，查看部署状态。

## 常见问题排查

### 问题1：前端无法连接后端

**检查步骤：**

1. 确认后端服务正在运行
```bash
ssh root@你的服务器IP
pm2 status
```

2. 确认防火墙已开放 3001 端口
```bash
sudo ufw status
```

3. 在浏览器控制台查看错误信息（F12 → Network）

4. 测试后端是否可以从外网访问
```bash
# 在本地电脑运行
curl http://你的服务器IP:3001/api/puzzle/games
```

### 问题2：CORS 错误

如果看到 CORS 错误，在 `server.js` 中确认 CORS 配置：

```javascript
app.use(cors({
  origin: '*',  // 允许所有来源
  methods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization']
}));
```

### 问题3：Vercel 部署失败

查看 Vercel 部署日志，确认 `vercel.json` 格式正确。

### 问题4：后端服务崩溃

查看 PM2 日志：
```bash
pm2 logs htp-backend --lines 100
```

重启服务：
```bash
pm2 restart htp-backend
```

## 性能优化建议

### 1. 启用 Gzip 压缩

在 `server.js` 中添加：
```javascript
import compression from 'compression';
app.use(compression());
```

### 2. 配置 Nginx 反向代理（可选）

如果想使用 80/443 端口，可以在服务器上安装 Nginx：

```bash
sudo apt install nginx
```

配置文件 `/etc/nginx/sites-available/default`：
```nginx
server {
    listen 80;
    server_name 你的服务器IP;

    location /api/ {
        proxy_pass http://localhost:3001/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

重启 Nginx：
```bash
sudo systemctl restart nginx
```

然后 `vercel.json` 中的端口改为 80：
```json
"destination": "http://你的服务器IP/api/:path*"
```

## 安全建议

1. **修改默认端口**：将 3001 改为其他端口
2. **配置 HTTPS**：使用 Let's Encrypt 免费证书
3. **限制访问来源**：在 CORS 中只允许你的域名
4. **定期更新依赖**：`npm audit fix`
5. **备份数据库**：定期备份 `data/behavior_analytics.db`
